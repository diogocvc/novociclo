# Design Vision
## Novo Ciclo

> O Novo Ciclo é uma experiência editorial construída para acompanhar uma jornada diária até a Copa do Mundo de 2030. Sua interface não deve competir com o conteúdo, mas organizá-lo de forma elegante, calma e extremamente legível. Este documento define os princípios visuais, estruturais e de experiência que devem orientar todas as decisões de design do projeto.

---

# Filosofia

Novo Ciclo não deve parecer uma plataforma de cursos, um blog tradicional ou um aplicativo de produtividade.

Ele deve transmitir a sensação de estar folheando um livro cuidadosamente diagramado, um diário de viagem ou uma revista editorial contemporânea.

O conteúdo é o protagonista.

A interface existe apenas para organizá-lo.

Cada decisão visual deve favorecer a leitura, a contemplação e o progresso natural da jornada.

---

# Princípios Fundamentais

## O conteúdo vem primeiro

Nenhum elemento visual deve chamar mais atenção do que o conteúdo do dia.

A interface precisa desaparecer.

O usuário deve sentir que está lendo um livro, não utilizando um software.

---

## Tipografia é a interface

A principal ferramenta de organização da informação será a tipografia.

A hierarquia será construída através de:

- tamanho
- peso
- espaçamento
- alinhamento

Não através de efeitos visuais.

---

## Espaço é parte do design

O vazio comunica.

As margens e os espaçamentos são tão importantes quanto o próprio conteúdo.

Sempre preferir:

- mais respiro
- menos elementos
- blocos bem separados

do que tentar ocupar toda a tela.

---

## Clareza absoluta

A relação entre os elementos deve ser compreendida instantaneamente.

Cada seção precisa possuir identidade própria através da organização espacial.

---

# Personalidade Visual

O sistema visual deve transmitir:

- Editorial
- Atemporal
- Humano
- Brasileiro
- Contemplativo
- Elegante
- Minimalista
- Calmo
- Generoso
- Organizado

---

# Referências

As referências utilizadas possuem uma característica em comum:

- tipografia extremamente dominante;
- poucos elementos por tela;
- blocos muito bem definidos;
- uso intencional da cor;
- grids limpos;
- grandes áreas de respiro.

A inspiração vem muito mais da linguagem editorial do que do universo das interfaces tradicionais.

---

# Estrutura da Página

Todas as páginas seguem praticamente a mesma organização.

```
────────────────────────────────────────────

Barra de Progresso

────────────────────────────────────────────

Conteúdo do Dia

────────────────────────────────────────────

Dias da Semana

────────────────────────────────────────────

Arquivo de Semanas

────────────────────────────────────────────

Footer

────────────────────────────────────────────
```

---

# Barra de Progresso

A barra de progresso representa a própria jornada.

Ela permanece sempre visível no topo da página.

Deve informar:

- dias já percorridos;
- dias restantes;
- percentual da jornada;
- contagem regressiva até a Copa de 2030.

Visualmente deve ocupar praticamente toda a largura da tela e funcionar como um elemento editorial, evitando a aparência de dashboards tecnológicos.

---

# Conteúdo do Dia

É sempre o protagonista da página.

Estrutura sugerida:

- data
- título
- subtítulo
- mídia principal
- conteúdo
- citações
- links
- reflexão final

Todo o conteúdo deve respirar.

Evitar blocos excessivamente densos.

---

# Navegação da Semana

Após o conteúdo principal, o usuário encontra os demais dias da semana.

Cada dia é apresentado como um cartão simples contendo:

- dia
- título
- pequeno resumo

O dia atual permanece destacado.

Os demais aparecem reduzidos.

A intenção é incentivar a continuidade da jornada.

---

# Arquivo Histórico

Abaixo da semana atual encontra-se o histórico completo.

As semanas aparecem recolhidas.

Ao expandir uma semana, surgem todos os dias daquela semana.

Cada dia funciona como um índice navegável.

A experiência deve lembrar o sumário de um livro.

---

# Footer

O rodapé deve ser extremamente discreto.

Contendo apenas:

- Manifesto
- Sobre
- Créditos
- Contato

---

# Sistema de Layout

## Desktop

Container máximo:

1200px

Largura ideal do conteúdo:

820–900px

A leitura nunca deve ocupar larguras excessivas.

---

## Tablet

Grid de quatro colunas.

---

## Mobile

Layout totalmente vertical.

Nenhuma sidebar.

Toda a navegação empilhada.

Prioridade absoluta para leitura.

---

# Grid

Desktop

12 colunas.

Tablet

4 colunas.

Mobile

1 coluna.

---

# Espaçamento

O sistema deve utilizar uma escala consistente.

```
8
16
24
40
64
96
128
```

Para separação entre grandes blocos utilizar preferencialmente:

- 64px
- 96px
- 128px

---

# Bordas

Sempre discretas.

Quando necessárias:

```
1px
```

em baixa opacidade.

---

# Cantos

Levemente arredondados.

Entre:

- 4px
- 8px

Nunca exagerar.

---

# Sombras

Não utilizar sombras decorativas.

A separação entre elementos acontece através do espaço.

---

# Tipografia

A tipografia é o principal elemento gráfico do projeto.

## Headline

Muito grande.

Impactante.

Inspirada em design editorial.

---

## Escala sugerida

H1

64–88px

H2

40–56px

H3

28–36px

Body

18–20px

Caption

14–16px

Sempre com excelente line-height.

# Uso das Cores

A maior parte da interface deve permanecer neutra.

O fundo creme deve dominar a experiência.

As cores aparecem apenas para:

- navegação;
- destaques;
- indicadores;
- componentes específicos.

Isso garante uma interface calma e duradoura.

---

# Componentes

## Botões

Grandes.

Muito padding.

Sem sombras.

Poucos estilos.

### Primário

Fundo sólido.

### Secundário

Contorno.

### Terciário

Somente texto.

---

## Cards

Pouca informação.

Muito espaço interno.

Tipografia grande.

Cor sólida.

---

## Ícones

Outline.

Traço fino.

Poucos.

Nunca ilustrativos.

---

# Imagens

Sempre grandes.

Poucas por página.

Preferência por:

- fotografia documental;
- fotografia histórica;
- natureza;
- pessoas reais;
- texturas de papel;
- imagens com aparência orgânica.

Evitar imagens excessivamente artificiais.

---

# Movimento

As animações devem reforçar a leitura.

Nunca chamar atenção para si mesmas.

Permitidos:

- fade
- slide
- accordion
- hover discreto

Evitar:

- bounce
- efeitos elásticos
- parallax exagerado
- animações decorativas

---

# Objetivo Final

O Novo Ciclo deve transmitir a sensação de uma publicação construída lentamente ao longo de cinco anos.

Cada visita representa um novo capítulo dessa jornada.

O usuário deve sair da interface com a impressão de ter passado alguns minutos folheando uma obra cuidadosamente editada, onde design e conteúdo trabalham juntos para criar uma experiência memorável, calma e profundamente humana.
