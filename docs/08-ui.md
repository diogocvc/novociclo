# 08 — Interface do Usuário

## Objetivo

Este documento define a interface do Novo Ciclo, especificando a estrutura, os componentes e o comportamento responsivo do web app.

A implementação deve seguir fielmente as definições de layout, componentes e comportamento descritas neste documento.

---

# Princípios de UX

## Hierarquia visual forte

O conteúdo principal deve ser identificado instantaneamente.

Nenhum elemento decorativo deve competir com a informação.

## Escaneabilidade elevada

O leitor deve conseguir extrair os principais acontecimentos do dia em segundos.

Títulos, subtítulos e chamadas devem funcionar como pontos de entrada visuais.

## Consistência no uso das cores

Cada cor possui um significado fixo.

Não deve existir variação de cor para o mesmo tipo de informação.

## Responsividade que preserva hierarquia

A prioridade da informação deve ser mantida em todos os tamanhos de tela.

O que é importante no desktop continua importante no mobile.

# Componentes

## Header

### Desktop

- Altura: 88 px.
- Largura total.
- Distribuição horizontal com espaçamento entre elementos.

```
ESQUERDA:                DIREITA:
LOGO                     Manifesto  Sobre  Créditos  Contato  [●]
```

### Mobile

- Logo à esquerda.
- Botão menu circular à direita.
- Navegação recolhida em menu hambúrguer.

### Logo

```
NOVO
CICLO
```

Tipografia bold, caixa alta, preta.

### Navegação

- Itens: Manifesto, Sobre, Créditos, Contato.
- Caixa alta.
- Tracking levemente expandido.
- Fonte: 12–14 px.

### Botão Menu

- Extremo direito.
- Círculo verde (~48 px).
- Ícone hambúrguer branco.

## Área Principal

### Desktop

Layout em duas colunas:
- Texto: ~45% da largura.
- Imagem: ~55% da largura.

### Coluna Esquerda

**Data:**
```

```
Fonte pequena, cor verde.

**Headline:**
```
Pequenas escolhas,
grandes mudanças
```
- Preto.
- Bold.
- Condensada.
- 72–84 px.

**Texto de apoio:**
Citação ou parágrafo introdutório em itálico ou com destaque tipográfico.

**CTA:**
```

```

- Imagem em formato 16:9.
- Bordas arredondadas.
- Sem sombra.
- Foto editorial de alta qualidade.

# Comportamentos e Estados

## Header

| Estado | Comportamento |
|---|---|
| Scroll down | Header recolhe ou mantém-se fixo (definir na implementação) |
| Mobile menu aberto | Overlay com navegação vertical |
| Link ativo | Destaque sutil no item correspondente |

## Countdown Banner

| Estado | Comportamento |
|---|---|
| Carregando | Skeleton loading com cores reduzidas |
| Normal | Exibe dados calculados |
| Erro | Exibe "--" nos valores numéricos |

## Cards Semanais

| Estado | Comportamento |
|---|---|
| Default | Fundo branco, texto preto |
| Ativo (hoje) | Fundo verde, texto branco |
| Hover | Leve escurecimento do fundo |
| Futuro (não publicado) | Opacidade reduzida (50%) |
| Mobile carrossel | Rolagem horizontal com snap |

## Accordion

| Estado | Comportamento |
|---|---|
| Recolhido | Mostra apenas semana e chevron |
| Expandido | Mostra dias da semana com links |
| Transição | Animação suave (fade/slide) |

# Grid e Layout

## Desktop

- Grid de 12 colunas.
- Container máximo: 1200 px.
- Conteúdo principal: largura ideal entre 820–900 px.

## Tablet (768–1024 px)

- Grid de 4 colunas.
- Espaçamentos reduzidos em 25%.
- Título ligeiramente menor (escala adaptativa).
- Cards semanais podem iniciar rolagem horizontal.

## Mobile (< 768 px)

- Layout em coluna única.
- Padding lateral: 16–24 px.
- Navegação empilhada.
- Prioridade absoluta para leitura.

# Princípio Final

A interface do Novo Ciclo não deve competir com o conteúdo.

Ela deve organizar, hierarquizar e apresentar.

O leitor deve sentir que está folheando um diário cuidadosamente editado, não utilizando um software.
